KAROKA — Guide complet du système de présence

Tableau de bord
Lecture des présences — vue quotidienne, hebdomadaire et mensuelle
Vue Jour
Liste de toutes les présences du jour sélectionné. Colonnes : arrivée/départ matin et après-midi, durée réelle, contrat théorique, écart.
Vue Semaine
Grille sur 7 jours (lun–dim). Cliquer sur l'en-tête d'une colonne de date pour ouvrir le détail complet de cette journée.
Vue S-Mois
Résumé du mois regroupé par semaine. Cliquer sur une cellule pour déplier le détail jour par jour de la semaine.
Vue Mois
Grille complète du mois calendaire. Filtrable par relevé de paie. Cliquer sur une cellule pour le détail du jour.
Contrôles de la barre supérieure
D
Sélecteur de date — Change la date de référence pour toutes les vues. Cliquer dessus pour ouvrir le calendrier.
R
Sélecteur de relevé — Filtre les vues Mois et S-Mois sur la période d'un relevé de paie défini dans l'administration. Visible uniquement en modes Mois/S-Mois.
🔍
Recherche — Filtre instantané par PRNO, nom, heure ou statut.
X
Exporter Excel — Télécharge le fichier presences.xlsx mis à jour avec toutes les données.

Statuts & Badges
Signification de chaque indicateur de présence
Complet Toutes les plages pointées (arrivée + départ matin et/ou après-midi) et durée réelle conforme au contrat.
Incomplet Pointage partiel (arrivée sans départ, ou l'inverse) ou durée réelle inférieure au contrat (écart négatif).
Absent Aucun pointage enregistré pour un jour normalement travaillé selon le code horaire de l'employé.
Férié Jour férié déclaré dans l'administration. Les absences ce jour ne sont pas comptabilisées.
Exclu Jour non travaillé selon le code horaire (week-end, jour off explicite). Aucun écart calculé.
Sans horaire Aucun code horaire affecté à cet employé — impossible de calculer la conformité.
Calcul du temps réel
Le temps est plafonné sur chaque plage : arriver en avance ou partir en retard ne donne pas de bonus. Une franchise de retard de 15 min à l'arrivée n'est pas décomptée ; au-delà, seul le dépassement compte (ex : 20 min de retard → 5 min déduites). Un départ anticipé est pénalisé dès la 1re minute. La tolérance d'admission d'un pointage hors plage est de ±1 h.
H
Codes horaires
Syntaxe pour définir les plages de travail d'un employé
Segment de travail
HHMMjourHHMM
Plage de travail d'un jour.
Ex: 0800lu1200
Pause
(HHMMjourHHMM)
Entre parenthèses — durée déduite.
Ex: (1200tj1300)
Exclusion de jour
(di) — tout le dimanche
(me:am) — matin seulement
(sa:pm) — après-midi
Jours disponibles
lu ma me je ve sa di
tj = tous les jours
Exemples concrets
// Lundi matin + après-midi seulement
0800lu1200;1300lu1700

// Lun–Ven 8h-17h (sam+dim exclus)
0800tj1200;1300tj1700;(sa);(di)

// Tous les jours 8h-17h avec pause déjeuner
0800tj1200;1300tj1700;(1200tj1300)

// Gardien de nuit (chevauchement minuit)
2000tj0600;(di)

// Mercredi : matin uniquement
0800tj1200;1300tj1700;(me:pm);(sa);(di)
Les segments sont séparés par ; — l'ordre n'importe pas. Utilisez le bouton Valider dans l'admin pour vérifier la syntaxe avant de sauvegarder.
A
Administration
Gestion des employés, horaires, relevés et jours fériés
Onglet Employés
1
Ajouter — Renseignez le PRNO (code unique, ex: pira) et le nom complet. Le code horaire est optionnel à cette étape.
2
Affecter un horaire — Cliquez sur ✏ Horaire dans la ligne de l'employé, saisissez le code et une date d'effet. L'horaire s'applique à partir de cette date.
3
Désactiver / Réactiver — Un employé désactivé ne génère plus de pointages. Il peut être réactivé ultérieurement.
4
Import CSV — Format attendu : prno,nom_complet,code_horaire (un employé par ligne).
Empreintes & PIN (ESP32)
E
Empreinte digitale — Chaque doigt enregistré sur le lecteur a un fingerprint_id. Liez-le à l'employé via le bouton + Lier de la colonne Empreinte.
P
Code PIN — Chaque code PIN enregistré a un pin_id. Liez-le via le bouton + Lier de la colonne PIN.
Onglet Relevés de paie
1
Créer — Choisissez le jour de paiement (ex: 20 → du 20 au 20 du mois suivant), le mois de début et un libellé optionnel. Le premier relevé est créé manuellement.
2
Clore — Clôturer un relevé actif génère automatiquement le suivant avec les mêmes paramètres.
3
Utiliser dans le dashboard — Le sélecteur de relevé (visible en modes Mois/S-Mois) filtre les données sur la période exacte du relevé.
Onglet Jours fériés
F
Ajouter un jour férié — Saisissez la date, un libellé et choisissez si le férié est fixe (même date chaque année) ou variable (date unique). Les absences les jours fériés ne comptent pas dans les statistiques.
Onglet Historique
H
Journal des pointages — Liste tous les pointages, du plus récent au plus ancien, avec la provenance (device) et le lieu.
🔍
Filtres — Recherche par nom ou PRNO, par date (yyyy/mm/dd) et par device (Telegram, Empreinte, Badge, PIN). Bouton Réinitialiser pour tout effacer.
📍
Device & lieu — Un pointage Telegram est affiché « À distance » ; un pointage par Empreinte / Badge / PIN est localisé au « Siège KAROKA » (lecteur physique). Les pointages antérieurs à cette fonctionnalité apparaissent comme Telegram.
C
Connexion & Accès
Identifiants et rôles utilisateur
URL
Adresse IP du serveur sur le port configuré (défaut 5000). Exemple : 192.168.1.10:5000
Identifiants
Définis dans .env via la variable RH_CREDENTIALS.
Format : user:pass,user2:pass2
Rôles disponibles
rh — Accès complet (dashboard + admin)
pdg — Accès identique
La session expire après 12 heures. Pour changer les mots de passe, modifiez la variable RH_CREDENTIALS dans le fichier .env et redémarrez le serveur.
